<h2 align="center">Act I &nbsp;—&nbsp; The Origin</h2>

<br/>

<p align="center"><i>
  "I didn't start with a plan. I started with a problem<br/>
  I couldn't stop thinking about."
</i></p>

<br/>
<br/>

<table>
<tr>
<td width="55%" valign="top">

It started the way most obsessions do — quietly. A broken script. A question about why something didn't work. A 2 AM rabbit hole that turned into a 2-year trajectory.

I never had a five-year roadmap. I had a screen, a compiler, and the stubborn belief that if I understood a system deeply enough, I could rebuild it better. So I did. Again. And again. And again.

**97 repositories later**, the pattern is clear: I don't build for demos. I build for production. For teams that need to ship on Friday and sleep on Saturday. For systems that don't page you at 3 AM because someone forgot an edge case.

Deep learning research, cloud-native APIs, full-stack platforms, developer tools — the domain changes but the standard doesn't. **Clean code. Solid docs. A working demo. Every single time.**

I document because I've been the new engineer reading someone else's mess. I teach because someone once taught me. I mentor because **the best systems aren't built by individuals — they're built by teams that trust each other's code.**
